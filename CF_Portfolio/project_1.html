Pipette Gravimetric Fixture
Brief

The objective of this project is to create a fixture that can effectively validate the precision and accuracy of both single-channel and eight-channel pipettes. Our approach involves utilizing a custom robot to ensure consistent and reliable results that align with the intended liquid handling capabilities of the pipettes. By meeting and exceeding the expectations of our customers, we strive to deliver a solution that instills confidence in the performance and reliability of the pipettes being validated.

Development:
  • Implement an acrylic housing to safeguard the analytical balance against disruptive air flow.
  • Mitigate vibration forces by utilizing separate tables to isolate the robot.
  • Develop a comprehensive test script that automates motion control, facilitates communication with the analytical balance, and ensures accurate data collection.
  • Create a dedicated protocol module to efficiently extract readings from the scale.
  • Establish clear PASS/FAIL criteria to streamline the validation process.
  • Prioritize user-friendliness to enhance ease of operation for operators.
  • Furnish customers with a calibration certificate for their pipettes, ensuring confidence and traceability in their usage.

Structure of Algorithm

  • The QC pipette script serves as the primary executor for conducting of various tests, encompassing function tests, volume tests, and multi-dispense tests, applicable to all pipettes.
  • Prior to running the QC pipette script, the calibration script enables operators to save positions for each labware on the deck.
  • The test script accommodates all pipettes, ensuring compatibility across different models and variations.
  • In the event of introducing a new pipette, a production staff member is responsible for adding the corresponding version to both the labware_config.json and the test_config.json files.

    • Three Python Libraries

  • The Data process module efficiently manages the processing of CSV data files for each pipette, as well as loading the configuration JSON files required for the test.
  • The Rig Function library consists of pre-defined functions specifically designed to handle various test functionalities.
  • The Robot Functions library encompasses essential robot functions and labware calibration functions. Examples include Move to Location, pick up tip, drop tip, trash tip, and more.
